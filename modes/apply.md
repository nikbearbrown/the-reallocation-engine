# Mode: apply -- Draft Application Support

Current status: draft workflow.

Use this only after `modes/oferta.md` has produced an evidence-based role
evaluation. This mode helps draft application answers; it does not decide
whether a role is worth pursuing.

## Required Context

- `modes/_shared.md`
- the saved evaluation under `data/ats/reports/`, if one exists
- the relevant Markdown CV under `resumes/`, if the student is using one
- `modes/RUN_LOG.md`

## Rules

- Use only facts from the posting, evaluation report, and CV.
- Do not invent projects, credentials, personal stories, work authorization, or
  contact details.
- If the application asks for a claim that is not supported by the CV, draft an
  honest adjacent answer or mark it as a gap.
- Log meaningful application milestones in `modes/RUN_LOG.md` or the tracker.

## Output

Provide copy-paste answers with a short evidence note for each answer:

```markdown
### Question
...

### Draft Answer
...

### Evidence Used
- Posting: ...
- CV/report: ...
```
